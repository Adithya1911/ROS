Creating and Setting Up a ROS Workspace

    Create an Empty Directory for the Workspace:

    First, create a directory for your ROS workspace. In this case, we'll name it ros1_ws. Run the following command in the terminal:

    mkdir ros1_ws

    Navigate into the Workspace and Create a src Directory:

    Change into the newly created workspace directory and create a src directory where all your executable code and packages will reside:

    cd ros1_ws

    mkdir src

    Initialize the Workspace:

    To initialize the workspace and install necessary dependencies, run the following command from within the workspace (not inside the src directory):

    catkin_make

    This command will generate two directories: build and devel, which are essential for managing the workspace's dependencies and build process.

    Building and Updating the Workspace:

    Whenever you update the src directory (e.g., adding new packages or modifying code), you need to rebuild the workspace using the following command:

    catkin_make

    This ensures that all updates are integrated into the workspace.

    Sourcing the Workspace:

    After building or updating the workspace, you must source the workspace setup file to your environment. This can be done with the following command:

    source ~/ros1_ws/devel/setup.bash

    Note: You must run this command every time you start a new terminal session unless you add it to your system's .bashrc file.

    Adding the Source Command to .bashrc:

    If you want to avoid sourcing the workspace manually each time, add the following line to your .bashrc file:

    echo "source ~/ros1_ws/devel/setup.bash" >> ~/.bashrc

    Ensure that this line is placed below the line that sources the ROS setup file (/opt/ros/noetic/setup.bash) in your .bashrc. This way, the ROS environment is fully set up each time you open a terminal.
