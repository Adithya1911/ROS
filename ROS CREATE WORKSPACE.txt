Creating and Setting Up a ROS Workspace

    Create an Empty Directory for the Workspace:

    First, create a directory for your ROS workspace. In this case, we'll name it ros1_ws. Run the following command in the terminal:

    mkdir ros1_ws

    Navigate into the Workspace and Create a src Directory:

    Change into the newly created workspace directory and create a src directory where all your executable code and packages will reside:

    cd ros1_ws

    mkdir src

    Initialize the Workspace:

    To initialize the workspace and install necessary dependencies, run the following command from within the workspace (not inside the src directory):

    catkin_make

    This command will generate two directories: build and devel, which are essential for managing the workspace's dependencies and build process.

    Building and Updating the Workspace:

    Whenever you update the src directory (e.g., adding new packages or modifying code), you need to rebuild the workspace using the following command:

    catkin_make

    This ensures that all updates are integrated into the workspace.

    Sourcing the Workspace:

    After building or updating the workspace, you must source the workspace setup file to your environment. This can be done with the following command:

    source ~/ros1_ws/devel/setup.bash

    Note: You must run this command every time you start a new terminal session unless you add it to your system's .bashrc file.

    Adding the Source Command to .bashrc:

    If you want to avoid sourcing the workspace manually each time, add the following line to your .bashrc file:

    echo "source ~/ros1_ws/devel/setup.bash" >> ~/.bashrc

    Ensure that this line is placed below the line that sources the ROS setup file (/opt/ros/noetic/setup.bash) in your .bashrc. This way, the ROS environment is fully set up each time you open a terminal.

CREATING THE PACKAGE 

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Navigate to the src Directory:

To create a new package, first navigate to the src directory within your workspace. In this case, the workspace is named ros1_ws:

cd ~/ros1_ws/src

Create the Package:

Use the catkin_create_pkg command to create a new package. The command syntax is as follows:

catkin_create_pkg package_name dependencies

Example:

catkin_create_pkg begin rospy turtlesim roscpp

    rospy: Allows you to create nodes using Python.
    roscpp: Allows you to create nodes using C++.
    turtlesim: Adds the turtlesim dependency to the package.

Understanding the Package Structure:

After running the command, a directory named begin (or your specified package name) will be created. This directory will contain:

    package.xml: Defines the packageâ€™s metadata and dependencies.
    CMakeLists.txt: Contains the build configuration for the package.

Building the Package:

After creating or updating the package, navigate back to the root of your workspace and run catkin_make to build the workspace. This step is crucial, especially after modifying any C++ (roscpp) files:

cd ~/ros1_ws

catkin_make

Sourcing the Workspace:

Once the workspace is built, ensure that you source the workspace setup file to apply the changes:

source ~/ros1_ws/devel/setup.bash

To create and organize ROS launch files, follow these steps:
1. Create a Directory for Launch Files

To organize your launch files, first create a directory named launch inside your ROS package directory.

'mkdir launch'

2. Create a Launch File

Within the launch directory, create a new launch file with the .launch extension. Use your preferred text editor to create and edit the file. For example, to create a launch file named my_launch.launch:

'touch launch/my_launch.launch'

Edit the my_launch.launch file and include the following XML structure:

<launch>
    <node name="node_name_1" pkg="package_name_1" type="filename_1" output="screen"/>
    <node name="node_name_2" pkg="package_name_2" type="filename_2" output="screen"/>
    <node name="node_name_3" pkg="package_name_3" type="filename_3" output="screen"/>
</launch>

Replace the placeholders with the appropriate values:

    node_name: Unique name for the node.
    package_name: The name of the ROS package containing the node.
    filename_with_extension: The executable file within the package (usually .py or .cpp).
    output: Typically set to "screen" to display output in the terminal.

3. Make the Launch File Executable
Make the launch file executable using the chmod command:

'chmod +x launch/my_launch.launch'

Run the Launch File

Finally, run the launch file using the roslaunch command:

'roslaunch package_name my_launch.launch'
